# S01: Consent Gate Popup

**ConsentGate popup infrastructure wired end-to-end — interface, presenter, view, container slot, UIFactory, boot gate, SceneSetup — 336/336 tests pass.**

## What Happened

Built the consent gate following the PlatformLinkPresenter pattern exactly. `IConsentGateView` has a single event (`OnAcceptClicked`) with no dismiss path. `ConsentGatePresenter` guards on a PlayerPrefs flag and exposes a UniTaskCompletionSource-based `WaitForAccept()`.

The key structural decision was hoisting `PopupManager` construction to the very top of `GameBootstrapper.Start()` — it only needs the `[SerializeField]` fields (`_viewContainer`, `_inputBlocker`) which are available at MonoBehaviour start. This lets the consent gate use the popup stack before any services are constructed.

`ConsentGateView` subclasses `PopupViewBase` for the standard bounce-in/scale-out animations. Link buttons call `Application.OpenURL("https://simplemagicstudios.com/play")`. SceneSetup now instantiates `ConsentGatePopup` prefab (logs a warning if prefab doesn't exist — same graceful fallback as other popups).

The test for no-dismiss-path uses reflection (`GetEvents()` on the interface) to mechanically assert exactly one event exists — prevents accidental addition of a close/skip path.
