# S01: Deck Panel Wiring

**Goal:** Wire the DeckPanel as a functional UGUI HorizontalLayoutGroup with one Button per tray slot, driven by existing RefreshSlot calls and firing OnTapPiece on tap.

**Demo:** Open InGame scene in play mode, start a level — deck panel shows buttons for each active slot. Tap a button → piece is placed (or rejected). Buttons update as pieces are placed.

## Must-Haves

- `InGameView` has a `_pieceButtonContainer` (RectTransform child of `_deckPanel`) with `HorizontalLayoutGroup` and `ContentSizeFitter`
- `InGameView.SetupDeckPanel(int slotCount)` creates `slotCount` Button children in `_pieceButtonContainer`, wires each to fire `OnTapPiece` with the piece ID in that slot
- `InGameView.RefreshSlot` updates the corresponding button: shows it (with piece label text) when a piece is in the slot, hides it when slot is null
- `_deckPanel` is no longer hidden in `Awake()` — it is active
- `PuzzleStageController.SpawnLevel()` calls `_inGameView.SetupDeckPanel(slotCount)` instead of `SpawnSlotButtons(slotCount)` (SpawnSlotButtons is removed in S02 — for S01, both can coexist or SpawnSlotButtons is made a no-op stub)
- `SceneSetup.cs` `CreateInGameScene()` updated: `DeckPanel` gets `HorizontalLayoutGroup`; `_pieceButtonContainer` child RectTransform added and wired to new `InGameView` field; legacy `DeckLabel` + `PlaceButton` children removed from DeckPanel
- SceneSetup re-run: `Assets/Scenes/InGame.unity` committed with updated layout
- All 347 existing tests pass (IInGameView interface unchanged)
