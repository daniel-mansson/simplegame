# S01: Deck Panel Wiring

**UGUI HorizontalLayoutGroup deck panel wired in InGameView — buttons driven by RefreshSlot, firing OnTapPiece on tap**

## What Happened

Added `SetupDeckPanel(int slotCount)` to `InGameView` — creates N UGUI Buttons inside `_pieceButtonContainer` (a `RectTransform` child of `DeckPanel` with `HorizontalLayoutGroup`). Each button's `onClick` reads the current slot contents and fires `OnTapPiece(pieceId)` — wiring into the existing presenter path with zero interface changes.

`RefreshSlot` extended to drive the corresponding button: shows it with `"Piece N"` label when a piece is assigned to the slot, hides it when null. The old `_deckLabel` slot-0 status label and `_placeButton` are removed — they were legacy placeholders.

`SceneSetup.CreateInGameScene()` updated to build `PieceButtonContainer` as a `HorizontalLayoutGroup` child of `DeckPanel`; wires `_pieceButtonContainer` on `InGameView`; no longer creates `DeckLabel` or `PlaceButton`. SceneSetup re-run cleanly — `[SceneSetup] InGame scene saved.` confirmed in editor log.

`PuzzleStageController.SpawnLevel()` calls `SetupDeckPanel(slotCount)` after `SpawnSlotButtons` (the overlay system — removed in S02).

## Deviations

None.
